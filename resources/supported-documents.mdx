---
title: "Document Types"
description: "Complete reference of all 30+ document types supported by the Atlas API and every field extracted from each document, organized by category."
---

The Atlas API automatically classifies and extracts structured data from a wide range of Indian financial and government documents. Each extracted field is returned with a `value` and a `confidence_score` (0–1 float) that indicates the model's certainty about the extraction. This page lists every supported document type and its corresponding fields, organized by category.

  <Accordion title="Identity Documents">
    ### AADHAAR

    | Field | Description |
    | --- | --- |
    | `name` | Full name as printed on the card |
    | `address` | Raw address string |
    | `formatted_address` | Normalized address object (`state`, `postcode`, `address_line_1`, `address_line_2`, `city`) |
    | `aadhaar_number` | 12-digit Aadhaar number; also includes a `redact_url` field — a pre-signed S3 URL to a redacted version of the original image |
    | `date_of_birth` | Date of birth |

    <Note>
      The `aadhaar_number` field includes a `redact_url` in addition to `value` and `confidence_score`. Use the `redact_url` to retrieve the image with the Aadhaar number masked. See the [FAQ](/resources/faq) for details on secure handling.
    </Note>
